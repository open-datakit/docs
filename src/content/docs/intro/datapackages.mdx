## What are datapackages?

An opendata.studio **datapackage** is a complete description of a data analysis.

A datapackage contains definitions for:

- Data (**resources** and **metaschemas**)
- Analysis (**algorithms**, **arguments**, and **containers**)
- Visualisations (**views** and **pages**)

Datapackages can be executed via our web interface or [command line tool](https://github.com/opendatastudio/cli), maintaining a reproducible record of the analysis performed which can then be readily published.

## Datapackage structure

opendata.studio datapackages are based on the open-source [Frictionless Data Package](https://specs.frictionlessdata.io/data-package/) specification.

At the most basic level, a datapackage is a git repository consisting of `.json` configuration files describing each element of a data analysis.

A datapackage might look as follows:

  - containers/ the container the code will execute in
    - ic50-container/

  - **views/** visualisations
    - fitGraph.json
    - fitGraph.py
  - pages/ defines the web app interface for this datapackage
    - main.json
  - datapackage.json
</FileTree>
